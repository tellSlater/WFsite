Website anatomy

A website like this one is made from three languages: HTML, CSS and JavaScript. HTML is used to write any text and separate it into paragraphs or sections. CSS is then used on top of the HTML code to specify styling for the page such as fonts, colors or backgrounds. Last but not least, JavaScript helps make the page more interactive - you can make any element on the page change based on any user input.

You can definitely write a beautiful, well working website without ever having to write a line of JavaScript. If you are planning to learn and use JavaScript, having previous knowledge with any other programming language will help immensely. HTML and CSS are not exactly programming languages and are easier to grasp.

Hello world

Writing your first "Hello world!" site is as easy as creating a .txt file, renaming the file to "(use any name here).html", opening the file with a text editor, writing "Hello world!" and then saving it. Here is what such a file looks like when opened with a web browser --> helloWorld.html.

Now if instead of writing just "Hello world!" in this file, I wrote "<h1>Hello world!</h1>" the resulting website would be this --> helloWorld2.html. I used HTML markup code to indicate that my text is an h1 headline so the font is now bigger in size.

Changing the color can be done by using CSS. This time I write two lines of code in my HTML text file. On the first line I use CSS to indicate that all h1 headlines should be blue: "<style>h1{color: blue;}</style>". The second line of code is the same line that was used before: "<h1>Hello world!</h1>". Opening my file in a web browser now results in --> helloWorld3.html.

Source for learning

I learnt these three languages over on a website called w3schools (w3schools.com). Their tutorials are packed with important knowledge to get you started and are well written. There are also exercises and an online platform where you can mess around with code. I suggest starting with their HTML tutorials and then heading over to CSS. You can create a complete site using these two languages. If you are planning on implementing an audio player or a sortable grid of content or any other interactive element, you will need to learn to use JavaScript as well.

Text editor

A text editor is necessary for typing code. Although an entire site could be written using Windows' notepad, it is not the best choice. Today there are smarter text editors designed to make coding faster and more efficient. Modern text editors will help by marking your code, autofill code, guess what you want to write, auto format your code and more. One of the most famous editors today is Virtual Studio Code, it is available for free and is easy to use. This text editor has a big community behind it and a number of extensions available to make programming easier. Here is a list of extensions I use with VS Code for writing HTML, CSS and JavaScript:

  1. HTML CSS support - Code suggestions/autofill
  2. Auto Rename Tag - Helps rename html start and end tags simultaneously
  3. Bracket Pair Colorizer 2 - Colorizes bracket pairs (duh!)
  4. Prettier - Code formatter
  5. Live Server - Lets you open the page you are editing and have it change live, every time you change the code (very helpful)
  6. Indent rainbow - Helps visualize the code indents
  7. Code Spell Checker - Checks spelling for the English language

Versioning

If you are taking over a large programming project such as building a website, it is highly recommended that you use a versioning program. A versioning program helps keep track of all the changes you make in the code and why you make them. It helps you keep a very analytical history of your project's progress. It also gives the advantage of being able to rollback to previous versions which is invaluable if you ever regret changes that you made.

Git is the most used versioning program nowadays (Apr 2021). By default it is terminal based - it has no graphical user interface and is operated via a terminal. If that scares you, there are programs such as GitKraken designed to offer a graphical user interface for Git.

Inspecting

You can inspect any webpage's code and files by right clicking on it and selecting "inspect". In the inspect menu you can read the page's HTML, you can read the active CSS styling rules on each page element and you can read any JavaScript program that runs on the page. This is very useful in many ways:

  1. Inspect your own page to better understand how your code behaves
  2. Inspect your favorite web page to get inspired by how it is coded and have a good example in mind
  3. Use the JavaScript Debugger to tune and correct your JavaScript code

Hosting

After writing your website all that remains is making it available on the internet. You can host your website using your own computer and network or you can pay for a hosting service. The limitations of hosting yourself are your network speed and the fact that you need to keep your computer busy all the time. Other than that, hosting from your own PC is a good way to get some first impressions and it is free. Hosting alone is enough to make your site available on the internet. Anyone will be able to visit your site by entering the server's public IP address in a browser's URL bar.

Domain name

Associating a name to the server's IP address is advised because a name is easier to remember and share with people. After doing so, anyone who types your domain name in the URL bar, will be redirected to your server's IP. You can get a free domain name associated to your chosen IP by visiting noip.com. Alternatively, you can pay for a more formal name with a .com or .org extension.
